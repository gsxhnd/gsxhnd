|---------|--------|------|
| 移动设备 | 320px - 640px | 单列布局、小字体、简化导航 |
| 小平板 | 640px - 768px | 2列网格、中等字体 |
| 平板 | 768px - 1024px | 2-3列网格、普通字体 |
| 笔记本 | 1024px - 1280px | 3-4列网格、正常字体 |
| 桌面 | 1280px+ | 4列网格、较大字体 |

## 字体大小响应示例

### 页面标题

```
h1 {
  font-size: 2rem;      /* 移动设备 */
  
  @media (min-width: 768px) {
    font-size: 3rem;    /* 平板及以上 */
  }
}
```

### 服务网格

```
services__grid {
  grid-template-columns: 1fr;           /* 移动: 1列 */
  
  @media (min-width: 640px) {
    grid-template-columns: repeat(2, 1fr);  /* 平板: 2列 */
  }
  
  @media (min-width: 1024px) {
    grid-template-columns: repeat(3, 1fr);  /* 桌面: 3列 */
  }
}
```

## 测试响应式设计的方法

### 1. 使用浏览器开发者工具

- 按 F12 打开开发者工具
- 点击设备工具栏按钮
- 选择不同的设备预设进行测试

### 2. 模拟的断点

- iPhone SE (375px)
- iPhone 12 Pro (390px)
- iPad (768px)
- iPad Pro (1024px)
- Desktop (1280px+)

### 3. 实际设备测试

在真实设备上测试是最可靠的方法:

- 手机 (竖屏 & 横屏)
- 平板 (竖屏 & 横屏)
- 桌面电脑

## 性能提示

1. **最小化重排 (Reflow)**
   - 避免过度的媒体查询
   - 合并相关的样式更改

2. **优化加载时间**
   - 使用 CSS 变量减少冗余
   - 利用 Tailwind CSS 的优化

3. **移动优先性能**
   - 减少默认样式大小
   - 仅为更大屏幕增加样式

## 常见问题

**Q: 为什么使用 min-width 而不是 max-width？**
A: 移动优先方法更高效，因为移动设备通常功能较弱，应首先为其优化。

**Q: 如何在 1280px 和 1920px 之间添加断点？**
A: 在 global.css 中的 :root 变量添加新断点，然后在 SCSS 中使用它。

**Q: 响应式设计会影响性能吗？**
A: 当编写良好时，响应式设计对性能影响很小。Tailwind CSS 会优化和最小化输出。
